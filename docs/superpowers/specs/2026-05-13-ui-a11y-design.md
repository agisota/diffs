## Context

been through a long UI-polish thread (last ~15 commits). Two recent fixes
(`31fb230`, `ddee880`) already mitigated the obvious traps: a global

keyboard-user discoverability, screen-reader semantics, and reduced-motion

## Bundle 1 — Defensive fixes (~30 LOC)

One atomic commit. Risk: near-zero (additive).

### B1.1 — Extend keydown guards
Both global keydown handlers (lines 2373 and 2385) currently bail only on
`INPUT` and `TEXTAREA`. Extend each guard to also bail when:

- `e.target.tagName === 'SELECT'` — pressing `j` / `k` inside an open
  `<select>` (pairs-sort, settings-default-sort) currently triggers viewer
  jumps if the viewer modal is open.
- `e.target.isContentEditable === true` — defends future rich-text inputs.

The new guard becomes:

```js
const t = e.target;
const tag = (t && t.tagName || '').toUpperCase();
if (tag === 'INPUT' || tag === 'TEXTAREA' || tag === 'SELECT' || (t && t.isContentEditable)) return;
```

Apply at both line 2373 (viewer handler) and line 2385 (help / global
shortcut handler — but keep the existing Escape-from-input behaviour for
toasts at line 2396, which is desired).

### B1.2 — Anchor → button for in-page actions
Topbar lines 489–490 use `<a href="#" onclick="...">` to open modals.
Replace with `<button class="topbar-btn">`. Wire normal
`addEventListener('click', ...)` in the JS init block instead of inline
`onclick`.

Add a matching CSS rule (placed near the existing `.tab` rule at line ~58):

```css
.topbar-btn {
  background: transparent; border: 0;
  color: var(--mute); font-size: 12px;
  cursor: pointer; padding: 0;
  font: inherit;
}
.topbar-btn:hover { color: var(--fg); text-decoration: underline; }
```

### B1.3 — `aria-label` audit on emoji-only buttons
Add `aria-label` to:

- `#help-modal` close `✕` button (line ~637)
- `#settings-modal` close `✕` button (line ~659)
- `#settings-clear` 🗑 button (line ~676) — already has text, but the icon
  is decorative; mark with `aria-hidden="true"` on the emoji span
- Batch-list delete `×` button (`.batch-del`, line ~1087)
- Staged-file `×` button (line ~880) — already has `aria-label='remove'`,
  verify and keep
- Viewer modal `vm-close` button (line ~289) — has text "Закрыть"; OK
- Toast dismiss `×` button (line ~828) — already has `aria-label='dismiss'`

Rule: any button whose accessible name comes only from an emoji or symbol
must have an explicit `aria-label`. Buttons with visible text are fine.

### B2.2 — Focus trap helper
Add a single helper near other modal helpers:

```js
function _trapFocus(modalEl) {
  const handler = (e) => {
    if (e.key !== 'Tab' || modalEl.hidden) return;
    const focusable = modalEl.querySelectorAll(
      'a[href], button:not([disabled]), input:not([disabled]), select:not([disabled]), textarea:not([disabled]), [tabindex]:not([tabindex="-1"])'
    );
    if (!focusable.length) return;
    const first = focusable[0];
    const last = focusable[focusable.length - 1];
    if (e.shiftKey && document.activeElement === first) {
      e.preventDefault();
      last.focus();
    } else if (!e.shiftKey && document.activeElement === last) {
      e.preventDefault();
      first.focus();
    }
  };
  modalEl.addEventListener('keydown', handler);
  return () => modalEl.removeEventListener('keydown', handler);
}
```

Wire on each modal-open path (returns a teardown function to call on close).

### B2.4 — `prefers-reduced-motion`
Single CSS block placed after the `:root` block:

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    transition-duration: 0.01ms !important;
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    scroll-behavior: auto !important;
  }
  .dropzone.dragging { transform: none; }
  .batch-card:hover, .pair-card:hover { transform: none; }
}
```

Note the `0.01ms` (not `0`) — chosen so JS `transitionend` events still
fire and don't deadlock state machines that wait for them. None exist in
this file today, but it's a safe default.

### B2.5 — `:focus-visible` rings
Add a global rule (after the button reset at line 42):

```css
button:focus-visible,
a:focus-visible,
input:focus-visible,
select:focus-visible,
textarea:focus-visible,
[tabindex]:focus-visible {
  outline: 2px solid var(--blue);
  outline-offset: 2px;
}
```

This replaces the (invisible) browser default outline on the dark palette.
`:focus-visible` (not `:focus`) avoids the outline appearing on mouse click.

### B2.6 — `aria-live` on toasts
The toast container (`#toast-wrap`) needs `aria-live="polite"` and
`aria-atomic="false"`. Errors will be announced; routine confirmations
won't interrupt because they're polite, not assertive. The container
markup is in the HTML body (search for `id="toast-wrap"`).

## Risks

- **Focus trap with hidden focusable elements** — if a modal contains a
  child that's `hidden` or `display: none`, it'll still match
  `button:not([disabled])` but `.focus()` will silently no-op. The helper
  handles this correctly because `Tab` falls through to the next match;
  worst case is a single tab keystroke moves focus to the next visible
  element. Acceptable.
- **`aria-modal="true"` and overlay click-to-close** — both modals close
  on backdrop click via inline `onclick`. Keep that behavior; `aria-modal`
  doesn't conflict.
- **`outline-offset: 2px` clipping inside scroll containers** — the events
  table is `overflow: hidden` on the wrapper. Focus rings on row buttons
  could clip. If observed during verification, drop the offset to `0`.
